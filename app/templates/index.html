How It Works

1

Scrape Intel

Pull from CISA, NVD, RSS feeds (TheHackerNews, AWS Security Blog, Unit42), Reddit, and abuse.ch for the latest threats, CVEs, and IoCs.

→
2

Generate Detections

Map threat intel to CloudTrail events. Auto-generate detection modules with LogQL queries, detection logic, severity ratings, and runbooks.

→
3

Sandbox Validate

Spin up a Kubernetes cluster with Loki, inject synthetic CloudTrail events, run detections, measure precision. Flag false positives before production.

→
4

Deploy Templates

Export production-ready detection templates for Splunk (saved searches), Sentinel (KQL), and Elastic (detection rules). Customize and deploy to your SIEM.

Deploy to Your Cluster

# Clone the detection-as-code repo
git clone https://github.com/jackiecloudsec/detection-as-code.git
cd detection-as-code

# Option 1: Local sandbox (kind + Loki + MinIO)
./sandbox/setup.sh
python3 sandbox/inject_cloudtrail.py --count 50
kubectl logs -n detections -l app=detection-engine -f

# Option 2: Production Helm deploy
helm upgrade --install detection-engine ./helm \
  --set loki.url=http://your-loki:3100 \
  --set alertmanager.url=http://your-alertmanager:9093/api/v1/alerts \
  --set slack.webhookUrl=https://hooks.slack.com/services/YOUR/WEBHOOK

# Export SIEM templates
curl https://your-instance/api/detections/brute_force/template/splunk
curl https://your-instance/api/detections/brute_force/template/sentinel
curl https://your-instance/api/detections/brute_force/template/elastic
